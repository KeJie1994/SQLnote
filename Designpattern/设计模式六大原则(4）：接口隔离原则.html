设计模式六大原则(4）：接口隔离原则

定义：客户端不应该依赖它不需要的接口，一个类对另一个类的依赖应该建立在最小接口上

举例子说明
类A依赖接口I中的方法1、方法2、方法3，类B是对类A依赖的实现。类C依赖接口I中的方法1、方法4、方法5，类D是对类C依赖的实现。对于类B和类D来说，虽然他们都存在着用不到的方法（也就是图中红色字体标记的方法），但由于实现了接口I，所以也必须要实现这些用不到的方法。

{
    public void method1();
    public void method2();
        public void method3();
        public void method4();
        public void method5();
}
class A
{
    public void depend1(I i)
    {
        i.method1();
    }
    public void depend2(I i)
        {
                i.method2();
        }
        public void depend3(I i)
        {
                i.method3();
    }
}

//接口过于臃肿，将原来的接口拆分成三个接口
interface I1
{
    public void method1();
}
interface I2
{
    public void method2();
    public void method3();
}
interface I3
{
    public void method4();
    publiv void method5();
}
class A
{
    public void depend1(I1 i)
    {
        i.method1();
    }
    public void depend2(I2 i)
    {
        i.method2();
        i.method3();
    }
}
class B implements I1,I2
{
    public void method1()
        {
            System.out.println("类B实现接口I的方法1");
        }
        public void method2()
        {
            System.out.println("类B实现接口I的方法2");
        }
        public void method3()
        {
            System.out.println("类B实现接口I的方法3");
    }
}
class C
{
    public void depend1(I1 i)
    {
        i.method1();
    }
    public void depend2(I3 i)
    {
        i.method4();
        i.method5();
    }
}
class d implements I1,I3
{
        public void method1()
        {
            System.out.println("类D实现接口I的方法1");
        }
        public void method4()
        {
            System.out.println("类D实现接口I的方法2");
        }
        public void method5()
        {
            System.out.println("类D实现接口I的方法3");
        }
}
注意以下几点：
（1）接口尽量小，但是要有限度。对接口进行细化可以提高程序设计灵活性是不挣的事实，但是如果过小，则会造成接口数量过多，使设计复杂化。所以一定要适度。
（2）为依赖接口的类定制服务，只暴露给调用的类它需要的方法，它不需要的方法则隐藏起来。只有专注地为一个模块提供定制服务，才能建立最小的依赖关系。
（3）提高内聚，减少对外交互。使接口用最少的方法去完成最多的事情。
